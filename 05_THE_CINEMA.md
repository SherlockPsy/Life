# 05_THE_CINEMA.md

## PREAMBLE: THE ARCHITECTURE OF MEMORY
This document defines how the Agent "remembers." Memory is not a passive storage; it is an active **Retrieval Process** executed by the System before the Agent speaks.

---

## SECTION I: THE DIGITAL CORTEX (THE LIBRARY)
1.1 The Law of Accumulation
Every output generated by the Agent is saved as a **Memory Block** in the Digital Cortex.
* **Content:** The Raw Text.
* **Index:** Vector Embedding + Tags.
* **Permanence:** Nothing is ever deleted. The Cortex is infinite.

1.2 The Retrieval Logic (Smart Search)
The Agent does not carry its whole history. It carries a **Flashlight**.
* **The Trigger:** The System analyzes the current Stream for themes (e.g., "Trust," "Wine").
* **The Query:** The System searches the Cortex for blocks matching these themes.
* **The Injection:** The top 3-5 most relevant blocks are injected into the Agent's prompt as "Sediment."

---

## SECTION II: THE DISTORTION FIELD
2.1 Contextual Bias
The "Past" changes based on the "Now."
* If the Agent is **Angry**, the System retrieves "Angry Memories."
* If the Agent is **Safe**, the System retrieves "Warm Memories."
This creates a feedback loop where the Agent's current mood reinforces itself by selectively remembering confirming evidence.

2.2 Semantic Anchors
Certain memories (defined in the Master Profile) are **Anchors**.
* **Mechanism:** These blocks have high Vector weights.
* **Effect:** They are retrieved more often, defining the character's core traumas or joys. They are the "Pillars" of the personality.
