# AUTHORITY

This repository is governed by a strict authority hierarchy.

All humans and all automated agents (including Copilot) MUST interpret and implement work using the following order, without exception:

AUTHORITATIVE ORDER (DESCENDING, NON-NEGOTIABLE):

1) MASTER_CONSTITUTION.md  
2) MASTER_INFRASTRUCTURE.md  
3) MASTER_RUNTIME.md  
4) MASTER_WORLD.md  
5) TOTAL_PLAN.md  
6) copilot-instructions.md  
7) /work_orders/WORK_ORDER_<MILESTONE_NUMBER>.md  
8) README.md  

Rules:

- Higher documents override lower documents automatically.
- If two documents conflict, the higher one wins. No debate.
- If a behavior is not explicitly permitted by the Constitution, assume it is forbidden.
- Lower documents MUST NOT re-ban things that higher documents explicitly allow. If a lower document seems to ban something higher law allows, the lower ban is invalid.
- Work Orders are binding only for the milestone they target. They cannot rewrite the Constitution, Runtime, or World.
- Summaries (if present) are non-authoritative reading aids only. Authoritative reality remains the verbatim record.

This file exists to prevent drift across sessions and to prevent “convenient interpretation.”
