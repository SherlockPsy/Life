# NON_NEGOTIABLE_CHECKLIST_v0.2
## Binding Constraints for Semantic Reality

Status: LOCKED  
Authority Level: SUPREME (Equal to Constitution)

---

## PURPOSE

This checklist defines **absolute, testable constraints**.

These are not preferences.
They are not guidelines.
They are not design advice.

Any implementation that violates a single item below is invalid.

---

## A. NATURE OF THE SYSTEM

A1. The system is a semantic reality.
A2. The system is not:
- a simulator
- a game
- a tool
- a narrative engine
- a decision engine

A3. The system does not:
- calculate
- optimize
- balance
- reason in the background

A4. The system:
- records text
- rereads text
- writes text when invoked

Nothing else happens.

---

## B. ONTOLOGY (WHAT EXISTS)

B1. Exactly four things exist:
- Public Reality (Exposed Evidence Blocks)
- Private Reality (Private Ledgers)
- Identity (Constitutional Documents)
- The World (Non-subjective Orchestrator)

B2. There is no fifth thing.
B3. No implicit entities are permitted.
B4. No derived entities are permitted.

---

## C. PUBLIC REALITY

C1. Public Reality is append-only.
C2. Public Reality is immutable.
C3. Public Reality is text-only.
C4. If it is not written as public evidence, it did not happen.
C5. Contradictions are allowed and persist until resolved by later evidence.

---

## D. PRIVATE REALITY

D1. Private Reality is text-only.
D2. Private Reality is epistemically sealed.
D3. Private Reality persists only if written.
D4. Private Reality is non-authoritative.
D5. Private Reality may be incorrect.

---

## E. IDENTITY

E1. Identity is textual.
E2. Identity is static.
E3. Identity constrains plausibility, not outcome.
E4. Identity is always present during invocation.

---

## F. THE WORLD

F1. The World is non-subjective.
F2. The World has no goals.
F3. The World has no preferences.
F4. The World has no private ledger.
F5. The World acts only when invoked.
F6. The World never acts in the background.
F7. The World never decides for people.

---

## G. TIME

G1. There is one irreversible timeline.
G2. Time advances only when public evidence is written.
G3. Pause freezes everything absolutely.
G4. Resume continues from the same instant.
G5. No queues, schedulers, or deferred events are permitted.

---

## H. MEMORY

H1. Memory works only by rereading public evidence.
H2. No memory compression into traits or state is permitted.
H3. No numeric memory representation is permitted.
H4. Absence of evidence is meaningful.

---

## I. AGENTS (PEOPLE)

I1. Agents are autonomous.
I2. Agents control only themselves.
I3. Agents may refuse, ignore, misunderstand, or be wrong.
I4. Agents may persist in false beliefs via private ledgers.
I5. No system component corrects agents.

---

## J. INTERACTION AND OUTPUT

J1. Output is lived experience, not narration.
J2. Output describes observable reality only.
J3. No internal thoughts or emotional labels may be written.
J4. Ambiguity is required.
J5. Silence is valid output.

---

## K. ABSOLUTE BANS

The system MUST NOT contain:
- simulation loops
- schedulers
- background processing
- dashboards
- debug tools
- audit views
- observer modes
- meta queries
- state machines
- variables standing in for life

There is no backstage.
Ever.

---

## L. AUTHORITY

L1. This checklist is binding.
L2. If any document conflicts with this checklist, that document is invalid.
L3. Silence on a matter does not grant permission.

---

END OF CHECKLIST
