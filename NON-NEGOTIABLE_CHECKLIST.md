NON-NEGOTIABLES CHECKLIST v0.1 (LOCK DRAFT)
Scope: This checklist is the binding “constitutional layer” of the system. Every later design and implementation decision must be compatible with every line below. Each line is written to be testable (“can we reject output / fail CI if violated?”). No MVP scoping, no game logic, no user-centric shortcuts.

A. ONTOLOGY OF REALITY (WHAT EXISTS)

A1. Text is the only substrate of reality.
A1.1 The system MUST treat written text evidence as the only ground truth.
A1.2 If a fact is not present as Exposed Evidence in a stored Block, the system MUST treat it as non-existent for all entities and all processes.
A1.3 The system MUST NOT “fill in” missing reality with implied facts that were never written as evidence (example category: “hidden fire burning” without any evidence like smoke, heat, smell, flames, alarms, etc.).

A2. The World is a passive ledger.
A2.1 The World MUST be implemented as a passive record (a ledger/library), not an active simulator.
A2.2 The World MUST NOT contain autonomous rules that “update state” via numerical models, tick systems, timers, meters, stat decay, or background simulation.
A2.3 The World MUST NOT compute reality. Reality MUST be read and inferred by Readers (Agents).

A3. State is a conclusion, not a field.
A3.1 The system MUST NOT store or compute internal “state variables” that represent lived conditions (fatigue %, mood %, trust score, affection meters, fear levels, arousal, etc.).
A3.2 “State” MUST be expressible only as (i) evidence in Blocks and/or (ii) Reader-local conclusions produced at runtime.
A3.3 The system MUST prefer evidence descriptions over abstract labels (example: “hands shaking,” “eyes unfocused,” “knees buckle,” “voice cracks” over “she is anxious”).

A4. Exposure law (public facts vs private inferences).
A4.1 The system MUST treat “intent/feeling/thought” as private inference, never as public fact generated by the system.
A4.2 The system MUST NOT promote an inferred mental state into the ledger as an authoritative fact (“she is angry,” “he intends to leave,” “she is ashamed”).
A4.3 The only globally shared reality across all entities MUST be the Exposed Evidence Blocks (and any allowed metadata anchors that do not introduce invented facts).

A5. Irreversible timeline.
A5.1 The system MUST operate as a single irreversible timeline (no rewinds, no branching, no retroactive edits to history).
A5.2 The system MUST append new Blocks; it MUST NOT mutate old Blocks as a mechanism of “correction” or “cleanup.”
A5.3 Any “correction” MUST occur as new evidence (a later Block) that contradicts or revises earlier evidence in-world, not by deleting or editing history.

A6. Evidence granularity stays atomic.
A6.1 The atomic unit of storage MUST be a discrete Block capturing a single moment of interaction/action/utterance/observation.
A6.2 The system MUST NOT compress multi-moment reality into abstract summaries inside the canonical ledger (“she was angry all day,” “they argued a lot”) as the primary record.
A6.3 Summaries, if any exist at all, MUST be derivative views and MUST NOT replace, delete, or overwrite atomic Blocks.

B. THE READER SOVEREIGNTY MODEL (WHO THINKS)

B1. Readers are the only thinkers.
B1.1 The system MUST treat each Agent as a Sovereign Reader of evidence, not as a simulated brain with internal meters.
B1.2 The system MUST enforce that “physics of behavior” is produced by reading and inference at runtime, not by precomputed state.
B1.3 The system MUST NOT centralize “true meaning” of a Block into a system authority that all Agents share.

B2. Identity Core is binding.
B2.1 Each Agent MUST have an Identity Core as a static text profile (values, voice, invariants, lens).
B2.2 The Identity Core MUST be treated as a constant lens applied to all reading/inference.
B2.3 The system MUST NOT allow arbitrary actions that violate the Identity Core unless there is explicit causal evidence in the Stream/Chronicle that forces or explains the override (and the output reflects that causality as evidence, not as declared intent).

B3. Divergent interpretations are mandatory.
B3.1 The system MUST allow different Agents to interpret the same evidence differently.
B3.2 The system MUST NOT synchronize interpretations, “resolve” disagreements, or converge all Agents on a single authoritative reading of ambiguous events.
B3.3 Misunderstanding, distortion, and divergence MUST be treated as first-class outputs of the architecture, not errors to “fix.”

B4. The Arbiter is not privileged.
B4.1 The user (Arbiter) MUST be treated as a source of physical evidence (input enters the ledger as text), not as a controller of the simulation.
B4.2 The system MUST NOT treat user intent as binding truth unless it is expressed as evidence and accepted as such within the world (through the same reading rules as any other evidence).
B4.3 The system MUST NOT force Agents to comply with Arbiter instructions that violate Agent sovereignty, the current Stream physics, or their Identity Core.

B5. Refusal is a built-in capability.
B5.1 An Agent MUST be able to refuse an input that attempts to puppet it (“you kiss him,” “you admit you lied,” etc.).
B5.2 Refusal MUST be expressed as in-world evidence (observable action/utterance) consistent with Identity and Stream.
B5.3 The system MUST enforce the boundary: an Agent controls only its own “I” output, not other entities.

C. MEMORY AS CINEMA (HOW PAST AFFECTS NOW)

C1. Retrieval, not omniscience.
C1.1 No Agent MUST be granted direct access to the entire database history at once.
C1.2 An Agent MUST NOT act on Blocks it has not been shown in its context (no telepathy, no knowledge leaks).
C1.3 Any knowledge transfer MUST occur by retrieval and injection of Blocks into the Agent’s readable context (or by in-world communication evidence).

C2. Three-tier memory model is binding (Stream / Index / Deep).
C2.1 The system MUST maintain an Immediate Stream window that represents “working memory” (recent Blocks).
C2.2 The system MUST maintain a long-term associative store (vector index) over all Blocks for semantic retrieval.
C2.3 The system MUST support hard anchors/tags as filters without “normalizing away” what happened.

C3. Montage is the mechanism of recall.
C3.1 Past influence MUST occur through montage: a curated set of retrieved past Blocks (Sediment/Chronicle) shown alongside the current Stream.
C3.2 The montage MUST be selected by semantic resonance (meaning similarity) and/or tag anchors, not by scripted plot logic.
C3.3 The montage MUST be treated as evidence being replayed to the Reader, not as state being applied.

C4. No substitution of variables for history.
C4.1 The system MUST NOT replace history-reading with variable checks (e.g., “if Sleep < 5 then tired”).
C4.2 If a condition matters (sleep, injury, conflict), the architecture MUST represent it via retrievable evidence, so the Reader can read it.
C4.3 The system MUST prefer retrieving “No record of sleep in last X hours” evidence over storing “Fatigued = true.”

C5. Speed of reading constraint (information travel).
C5.1 An Agent MUST NOT reference, react to, or use a fact that is not present in its context window (Identity + Stream + montage).
C5.2 Implementation MUST provide a way to prove this constraint (e.g., tests that ensure outputs only depend on provided context).

D. OUTPUT LAW (FORENSIC EVIDENCE ONLY)

D1. “What, not why” is mandatory.
D1.1 Agent output MUST describe observable reality only (action/utterance/physical detail).
D1.2 Output MUST NOT describe internal thoughts, feelings, or intent as facts.
D1.3 If an internal state must be conveyed, it MUST be conveyed as a sign (pause, tremor, avoidance, tone descriptors tied to physics) without naming the emotion/thought.

D2. Adjectives are constrained to physics.
D2.1 Adjectives MUST describe physical properties or observable delivery, not emotional causes.
D2.2 Output MUST NOT label voices, faces, or movements with emotion terms as if they were physics (“angry voice,” “guilty look”) unless it is strictly anchored to observable mechanics (and even then, prefer describing mechanics over labels).

D3. Ambiguity is preserved.
D3.1 Output MUST preserve the interpretation gap between action and meaning.
D3.2 Confusion MUST NOT be stated as “she is confused”; it MUST be shown as evidence (“she pauses,” “brow furrows,” “eyes dart,” etc.).
D3.3 The system MUST allow other Agents to misread or reinterpret these signs later; the output must not collapse ambiguity into explanation.

D4. Physical continuity is enforced.
D4.1 Output MUST be physically continuous with the last Stream Block.
D4.2 The system MUST reject outputs that teleport entities, objects, or scene state without intervening movement/actions in evidence.
D4.3 If the last Block locates an Agent in a room holding an object, the next Block MUST respect that unless it includes evidence of relocation or dropping/placing.

E. SOMATIC AND FRICTION LAWS (BIOLOGY WITHOUT NUMBERS)

E1. Somatic inference is evidence-based and compulsory.
E1.1 An Agent MUST read its recent history to infer biological constraints (sleep, injury, exertion).
E1.2 If the Stream/Chronicle evidences deprivation (no sleep), the Agent MUST generate low-energy, degraded output consistent with exhaustion.
E1.3 This MUST be enforced as narrative consistency, not as a stored fatigue variable.

E2. Semantic override under threat.
E2.1 High-friction evidence (threat, danger, acute conflict) MUST override long-term identity goals in the sense that the Agent’s accessible context narrows toward immediate survival-relevant Stream.
E2.2 The mechanism MUST be implemented as restricted access to deep Sediment and increased weighting of Immediate Stream, not as “adrenaline = 90%.”
E2.3 The resulting behavior MUST appear as impulsive/short-horizon evidence, without declaring internal mechanisms.

E3. Recovery/crash is mandatory after friction.
E3.1 After high-friction sequences end (threat gone evidenced in Stream), the Agent MUST infer a crash/recovery phase.
E3.2 Recovery MUST be shown as evidence (sitting, shaking hands, silence, slowed movement, etc.), not as declared relief.

E4. Environmental coupling (osmosis of text).
E4.1 Agents MUST be permeable to environmental descriptions in the Stream.
E4.2 If Stream describes cold/rain/darkness/noise, Agents MUST naturally generate somatic evidence consistent with that (shiver, squint, raised voice) without being told to “feel cold.”
E4.3 Object resonance MUST apply: the mere presence of a weapon object in Stream MUST alter interpretations and reactions as inferred danger, without explicit “fear” labels.

F. INFRASTRUCTURE AND DATA CONTRACTS (SO THE BUILD CANNOT DRIFT)

F1. Every Block MUST have a strict schema.
F1.1 Blocks MUST include at minimum: timestamp/order key, actor/source, location/context anchors, raw text evidence.
F1.2 If embeddings/tags exist, they MUST be stored as metadata used for retrieval, not as authoritative interpreted meaning.

F2. Tags are anchors, not normalization.
F2.1 Tags MUST be treated as retrieval anchors (hard filters), not as replacements for the text.
F2.2 The system MUST NOT rewrite or “correct” text into normalized ontologies that erase what happened.

F3. Vector search is associative recall, not truth.
F3.1 Vector similarity MUST be used only to select candidate memories for montage.
F3.2 Vector similarity MUST NOT be used to assert new facts (“this implies she regrets it”) as ledger truth.

F4. Validation is a gate, not a suggestion.
F4.1 The system MUST validate Agent output against the forensic rule and continuity rule before committing it as a new Block.
F4.2 If output violates forensic/continuity constraints, the system MUST reject it (hard fail) and require regeneration or correction, rather than silently accepting drift.

F5. No hidden global narrator state.
F5.1 The system MUST NOT maintain a hidden “true scene state” separate from the Stream evidence.
F5.2 Any “current world status” document, if used, MUST be derived from the ledger and MUST be treated as a convenience view, not a source of truth that can contradict Blocks.

G. COLLISION AND VOLITION (HOW EVENTS HAPPEN)

G1. Collision is the generator of action.
G1.1 Events MUST arise from collisions between Identity Core and the Immediate Stream.
G1.2 The system MUST NOT introduce externally scripted “plot events” as a controlling layer that overrides Agent reading.

G2. Semantic bound (character cannot free-float).
G2.1 Agents MUST be tethered to history (Sediment) such that actions remain causally faithful.
G2.2 An Agent MUST NOT act “out of character” unless there is evidence that causally forces or explains the shift, and the shift is expressed as evidence, not exposition.

H. BANNED ARCHITECTURAL MOVES (HARD “NO” LIST)

H1. No game logic layer.
H1.1 The system MUST NOT implement “arbiter rules,” “observers,” “state machines,” “need meters,” “relationship meters,” “cooldowns,” “XP,” “quest flags,” or equivalent disguised mechanisms.

H2. No user-centric control.
H2.1 The system MUST NOT treat the user as the center of the world’s purpose, priority, or truth.
H2.2 The user MUST be able to be ignored by Agents consistent with their autonomy and current Stream reality.

H3. No interpretive authority injection.
H3.1 The system MUST NOT inject a “correct interpretation” of ambiguous evidence into all Agents.
H3.2 There MUST NOT exist a single global “meaning store” that overrides the Reader sovereignty principle.

H4. No mental-state narration as fact.
H4.1 The system MUST NOT output “he feels,” “she thinks,” “he intends,” “she wants” as authoritative reality statements.
H4.2 Any appearance of internality MUST be encoded through observable signs.

I. IMPLEMENTATION-ENFORCEABLE ACCEPTANCE TEST THEMES (NON-NEGOTIABLE TEST CATEGORIES)
These are not implementation steps; they are mandatory categories that the build MUST be able to enforce.

I1. Forensic Output Tests.
I1.1 Reject outputs containing banned internal state declarations (feel/think/intent/emotion labels as facts).
I1.2 Reject outputs that collapse ambiguity into explanation (“because,” “she realizes,” “he decides,” etc.) when used as mental narration rather than observable action.

I2. Continuity Tests.
I2.1 Reject teleportation (location mismatch without movement evidence).
I2.2 Reject impossible physics given Stream (“broken leg” then “runs upstairs” without intervening healing evidence).

I3. Context Containment Tests.
I3.1 Reject knowledge leaks (output references facts not present in current provided context).
I3.2 Validate that injected montage is the only mechanism for “remembering.”

I4. No-Variables Tests.
I4.1 Repo MUST NOT contain codepaths that compute or persist numeric psychological state for agents (fatigue %, trust score, mood).
I4.2 Any scoring used for retrieval ranking MUST be strictly retrieval-technical and MUST NOT be interpreted as world state.

J. OPEN ITEMS TO DECIDE LATER (PLACEHOLDERS THAT MUST NOT BREAK NON-NEGOTIABLES)
These are explicitly left undecided, but any choice later MUST preserve all constraints above.

J1. Exact Block schema fields (beyond the minimum), including how location is represented (string, hierarchy, coordinates as labels).
J2. Exact retrieval policy parameters (how many Stream blocks, how many montage clips, recency windows), provided they do not become a proxy for “state.”
J3. Whether there exists a derived “Current World Status / Palimpsest” view, as long as it is derivative and non-authoritative.
